We also added a feature called asynchronous tool calling. These models are very fast if you're just doing casual conversation, but sometimes when you go off to do a tool call, that can take a long time. If you're just waiting for the tool call to come back to continue the conversation, you sometimes get stuck in an awkward silence with the user, and sometimes the user wonders what's going on because the system almost feels dead or broken. We enabled this asynchronous tool calling capability where agents can basically continue the conversation with the customer when the tool call is being executed in the background. When the tool call results come back, the agent incorporates that information into the dialogue to continue the conversation with the customer in a very natural way.

Another feature we added is the ability to add text into the speech input as well. So it's not just speech to speech; it's actually speech and text to speech and text. Even though we say these are voice-first experiences, having text capabilities is helpful for many use cases. For example, if you want the agent to say something before the customer says anything, you can use the text input from the developer side to prompt the model so the agent will start speaking without waiting for user speech input. You can also use that capability to handle touchpad input so that key presses can be converted into text as input into the model.

The benefit of this is that all of these are seamlessly incorporated into the model as if it's just one thing. There's never a loss of context or switch of context when you have different modality of input. It's all the same context information available to the model.
